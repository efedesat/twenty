# "Generate Quote" + "Approve Quote" — native Twenty workflows (no AI)

Two record-action buttons, each persisting a real record (no paused/dangling workflow state).
Built and verified live on the prebuilt `twentycrm/twenty:latest` image. The deterministic math is
`build/quote_logic_function.js` (mirrors `build/quote_engine.py`, which is the CLI reference/fallback).

**State machine (Quote.status):**
`Pending Approval` → (human clicks Approve Quote) → `Approved` → `Sent` → `Accepted`/`Declined`.

**Prereqs (all applied):** `LOGIC_FUNCTION_TYPE=LOCAL` on server+worker; Rate Card / Quote Line
objects; `enquiryType`; 8 rate cards; Quote status options incl. `Pending Approval` + `Approved`;
Quote `pricingSnapshot` TEXT field.

---

## Image-specific gotchas (learned the hard way — read before rebuilding)

1. **The manual trigger does NOT expose the triggering record's fields to the Code-step variable
   picker** on this image (the source says it should; the prebuilt image lags). Workaround:
   **re-fetch the Enquiry via its own Find Records step** and map from that. Trigger fields *are*
   available in Find-Records *filters* as `{{trigger.<field>}}` — e.g. `{{trigger.id}}`,
   `{{trigger.hotelId}}`, `{{trigger.companyId}}`.
2. **Relation filters match by Id, not Name.** Use field `Hotel → Id` / `Account → Id`, operand
   **Is** (matching `→ Name` with CONTAINS against an id silently returns nothing).
3. **Code-step inputs must map to the whole record** — use the step's `.all` (or `.first`), NOT
   `.first.id`. The function reads record fields, so passing an id string makes it throw
   "no rate card…".
4. **The Code step's output schema is generated by a Test run.** Selecting the Code step as a value
   source opens its Test screen; enter sample JSON for each param to produce a result, which exposes
   the output fields for mapping. Re-pasting code **resets the test inputs** — re-enter all of them,
   as raw JSON values (not field names).
5. **Currency caveat (door open):** Corporate Agreements carry a rate but no currency, so a
   SEK-negotiated rate gets applied as-is to a EUR/DKK/NOK hotel. Harmless for a SEK-hotel demo;
   add a currency guard before relying on it cross-currency.

## Workflow 1 — "Generate Quote"  (manual record action on **Enquiry**)

1. **Trigger — Manual**, object **Enquiry**, availability **Single record**. Name `Generate Quote`.
2. **Find Records → Rate Card**: `Hotel → Id` **Is** `{{trigger.hotelId}}`, limit 1.
3. **Find Records → Corporate Agreement**: `Account → Id` **Is** `{{trigger.companyId}}` **and**
   `Status` **Is** `Active`, limit 1.
4. **Find Records → Enquiry**: `Id` **Is** `{{trigger.id}}`, limit 1. *(re-fetch self — gotcha #1)*
5. **Find Records → Quote**: `Enquiry → Id` **Is** `{{trigger.id}}`, **limit 100**. *(for versioning)*
6. **Code step**: paste `build/quote_logic_function.js`. Map the 4 inputs to each find's **`.all`**:
   `enquiry`←(4), `rateCard`←(2), `agreement`←(3), `existingQuotes`←(5).
   Test with sample JSON (see gotcha #4) to expose the output schema.
7. **Create Record → Quote**:
   - `name` = `{{trigger.name}} – Quote v{{step6.version}}`
   - `enquiry` = `{{trigger.id}}` · `version` = `{{step6.version}}` · `status` = **Pending Approval**
   - `totalValue` = `{{step6.total}}` · `dayDelegateRateOffered` = `{{step6.ddr}}`
   - `pricingSnapshot` = `{{step6.snapshot}}`  *(full raw calc for audit/history)*
8. **Create Record → Quote Line (DDR)**: `quote`=created Quote, `lineType`=**Day Delegate**,
   `quantity`=`{{step6.ddrQty}}`, `unitRate`=`{{step6.ddrUnit}}`, `lineTotal`=`{{step6.ddrTotal}}`,
   `basis`=`{{step6.ddrBasis}}`.
9. **Create Record → Quote Line (Rooms)**: `quote`=created Quote, `lineType`=**Room Night**,
   `quantity`=`{{step6.roomsQty}}`, `unitRate`=`{{step6.roomsUnit}}`, `lineTotal`=`{{step6.roomsTotal}}`,
   `basis`=`{{step6.roomsBasis}}`.
10. **Publish / Activate.**

→ Each click creates a new **Quote (Pending Approval)** at the next version + 2 itemised lines, with
the full calculation stored in `pricingSnapshot`. Persisted immediately; nothing waits on a human.

## Workflow 2 — "Approve Quote"  (manual record action on **Quote**)  — no Code step

1. **Trigger — Manual**, object **Quote**, availability **Single record**. Name `Approve Quote`.
2. **Update Record → Quote**: record `{{trigger.id}}`, set `status` = **Approved**.
3. **Publish / Activate.**
